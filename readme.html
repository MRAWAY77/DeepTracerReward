DeepTrace Explorer

Local dataset visualizer and lightweight zero-shot video-VLM benchmark. It joins every real and generated clip with metadata, expert fake-trace annotations, model responses, deterministic localization scores, and reference-text diagnostics.

Run the visualizer

conda activate deeptracereward
python app.py

Open http://127.0.0.1:8000. Use python app.py --port 8080 if port 8000 is occupied. The visualizer itself uses only the Python standard library; inference dependencies are separate.

The fake collection spans Pika 1.5, Kling 1.0/1.5, Gen-3, Mochi 1, MiniMax-Video-01, and Sora. Real controls come from LLaVA-Video-178K and were clipped to match the fake-video length distribution. The official unique-video split is 5,308 train, 664 validation, and 664 test.

Why the annotations are valuable

This is substantially richer supervision than a binary real/fake label. An expert trace can identify what looked fake, where it appeared, and when it began and ended. A generated video can have multiple traces: 753 videos have two or more, with up to six traces on one video.

  • Experts performed frame-by-frame inspection in Labelbox and supplied temporally aware boxes, onset/offset, structured artifact tags, and natural-language descriptions.
  • Ambiguous cases were resolved collaboratively by majority consensus.
  • Each annotated video was reviewed by one or two additional annotators for cross-validation and bias reduction.
  • The source videos were manually filtered for high visual quality and meaningful motion, avoiding trivially degraded clips.
  • The released explanations were GPT-4-standardized to reduce linguistic inconsistency.
Important quality caveat. “Expert annotated” does not mean complete or objective ground truth. Explanation coverage is 62.7%; raw category tags are missing on many released rows; perceived fakeness is inherently subjective; and the paper does not report a conventional inter-annotator-agreement coefficient. The release is best described as rich, quality-controlled human-perception supervision, not flawless forensic truth.

Why this is a challenging benchmark

The videos were selected to be realistic and motion-rich rather than obvious failures. A system must distinguish subtle human-perceived anomalies from normal motion, then align an explanation with a small spatial region and temporal interval. The full zero-shot results reinforce the paper's reported difficulty gradient: classification is attainable for the strongest compact control, while precise explanation overlap and localization remain much harder.

Object distortion dominates the normalized taxonomy (2,157 mapped occurrences), followed by sudden blurring (719), trajectory anomalies (617), and merging (434). This imbalance means overall performance can hide weaknesses on rare disappearance, redundant-object, and background-distortion cases; use the visualizer's category and source filters for qualitative auditing.

Interpretation by model

  • Qwen2.5-VL-3B is the strongest usable lightweight baseline. It gave balanced enough transfer to exceed chance, produced concrete text for 661 samples, and supplied valid boxes/times for 189 fake videos. However, 66.6% classification and low 13.8 IoU leave a large gap to reliable forensic grounding.
  • VideoLLaMA3-2B has a strong REAL bias. It predicted REAL 580 times, explaining its 85.2% real accuracy but only 10.5% fake accuracy. All 664 responses triggered at least one schema/semantic warning, and it produced no valid fake-video boxes. Fluency here must not be mistaken for correctness.
  • SmolVLM2-500M confirms the lower capacity limit. Its label compliance, classification, grounding, and explanations are inadequate for this task, so it is retained only as a negative control.
  • Marlin-2B is an adapter/output failure, not a scientific model result. 650 rows returned text with no recognizable REAL/FAKE label and 651 rows were recorded as runtime/parse errors. Its nominal 1.054% score should not be compared with the other models until the adapter is repaired and rerun.

python inference/evaluate_explanations.py predictions/qwen2.5-vl-3b-v2-test.jsonl \

Repeat with videollama3-2b and marlin-2b. Classification/localization evaluation and explanation evaluation are separate because they answer different questions.

Interpretation and implementation caveats

  • The nine dashboard categories are heuristic mappings from noisy, multi-label release tags; they are exploratory, not a lossless official relabeling.
  • Never split or count by annotation row: repeated rows from one fake video would leak the same video across evaluation units.
  • Automatic explanation similarity measures reference agreement, not visual faithfulness. A model can mention the right artifact without actually grounding it in the video.
  • The latency bell curve describes runtime distribution. Classification accuracy is categorical and should be read through confusion counts and per-source bars.
  • Predictions are loaded from JSONL automatically. Restart the server after changing Python/HTML; newly appended prediction rows are discovered without rebuilding the dataset.

Completed Track-B spatial-localization benchmark

The final laptop-compatible shortlist is TruFor and IML-ViT. Both completed all 664 test videos without runtime errors using the same pretrained, two-frame-per-video protocol. These models were trained to find still-image manipulation fingerprints—not movement-centric human-perceived video traces—so the experiment measures out-of-domain transfer.

How to read these results

  • Neither model is a useful fake-video classifier without adaptation. TruFor predicted REAL for 640 of 664 videos; the near-zero fake accuracy and high real accuracy reveal this bias. Reporting only overall accuracy would make that failure less obvious.
  • TruFor is the stronger overall Track-B transfer baseline. It has the better classification balance, box IoU, and temporal distance. Its 13.962 IoU is still low, showing that a generic manipulation heatmap rarely aligns closely with the expert's perceived-motion box.
  • IML-ViT has slightly lower box-centre distance but poorer overlap. A prediction can be spatially near the expert region while having the wrong size or shape, so distance and IoU must be read together.
  • Temporal localization is derived and sparse. Each model sees two uniformly sampled frames; the timestamp of the strongest frame becomes its predicted onset. The normalized time distance is therefore a coarse transfer baseline, not native video temporal reasoning.
Visualizer support. Select trufor or iml-vit in Explore or Model performance. The detail view draws the expert box in red and the strongest model box in blue, then displays per-video IoU and temporal distance. Use SAME/DIFFERENT filters to expose correct labels, REAL-biased failures, and cases where localization overlaps despite an incorrect classification.

DeCLIP and MVSS-Net remain excluded because their official pinned environments conflict with the required Python 3.14 stack. They were not replaced with unverified ports merely to enlarge the benchmark table.
